# ESP32 → ESP32-S3 Migration Plan

## Status (2026-07-06)

**All migration work is complete.** Build succeeds, all source changes applied and verified.

### Completed work

- All build system targets migrated to `xtensa-esp32s3-espidf`
- GPIO pin re-assignment: STEP→GPIO21, ADC→GPIO4 (ADC1_CH3)
- Memory map constants updated for S3 (DRAM, IRAM, flash cache, UART0_BASE)
- RMT chunk assertion added (≤ 192 for 384-word shared RAM)
- Panic handler chain placed in IRAM (`.iram1`) for crash dump reliability
- All scripts updated to `xtensa-esp32s3-espidf` target
- BLE coexistence preference dead code removed (GR-4)
- Boot crash (LL-019) fixed: `logger::init()` moved before `disable_wdt()`
- Stale GPIO34 references cleaned up in `adc.rs` diagnostic and `docs/refs/project.md`
- `cargo check` — 0 errors, 0 warnings

### Remaining work (hardware validation — requires physical ESP32-S3)

- Flash to actual ESP32-S3 dev board via USB Serial/JTAG (`/dev/ttyACM0`)
- Run smoke tests per Phase 7 validation table
- Check stack watermarks for all threads
- Verify ADC calibration against known reference voltage
- 30-minute stability soak

### Key findings during migration

1. **ADC pin**: GPIO34 not an ADC pin on S3 (ADC1 pins are GPIO1-10 only). Migrated to GPIO4 (ADC1_CH3).
2. **IRAM section**: S3 linker script accepts `.iram1` for IRAM placement, NOT `.iram0.text` (which is the OUTPUT section name, not input).
3. **Panic handler in flash bug**: `__wrap_esp_panic_handler` and its call chain (do_backtrace, CrashWriter, is_sane_sp, etc.) must be in IRAM via `#[link_section = ".iram1"]` for crash dumps to appear after hardware exceptions.
4. **WDT + logging**: `disable_wdt()` must not call `log::warn!()` because it runs before `logger::init()`.

# ESP32 → ESP32-S3 Migration Plan

## Summary

Port the existing Rust/ESP-IDF v6 firmware from classic ESP32 (Xtensa LX6 dual-core, QFN48) to ESP32-S3 (Xtensa LX7 dual-core, QFN56). The S3 adds BLE 5.0, USB Serial/JTAG, split I/D cache, PIE SIMD extensions, and removes Bluetooth Classic. **Key blocking finding: GPIO25 does not exist on S3** — the STEP motor pin must be relocated.

### Chip comparison highlights

| Feature | ESP32 | ESP32-S3 |
|---|---|---|
| CPU | Xtensa LX6 dual-core @ 240 MHz | Xtensa LX7 dual-core @ 240 MHz (higher IPC, HW divide) |
| SRAM | 520 KB | 512 KB |
| DRAM | ~320 KB | ~360 KB |
| IRAM | ~192 KB | ~160 KB |
| Cache | 64 KB unified | Split I$ (16–32K) + D$ (32–64K) |
| RMT RAM | 512 × 32-bit (8 ch dynamic) | 384 × 32-bit shared (4 TX + 4 RX fixed) |
| USB Debug | External UART bridge only | Built-in USB Serial/JTAG (CDC-ACM + JTAG) |
| Bluetooth | 4.2 BR/EDR + BLE | 5.0 LE only (no Classic) |
| BLE PHY | 1 Mbps | 1 Mbps, 2 Mbps, Coded (125/500 Kbps) |
| WiFi | 802.11b/g/n, WPA2 | Same + WPA3, FTM, antenna diversity |
| ADC | 2 × 9 ch, 200 kSPS, 0–3900 mV @ DB_11 | 2 × 10 ch, 100 kSPS, 0–2900 mV @ DB_12 |
| GPIO | 34 (0–33, 25 exists) | 45 (0–21, 26–48, **25 absent**) |
| Strapping pins | 0, 2, 5, 12, 15 | 0, 3, 45, 46 |

### Phase 3: Source code — memory map constants

All addresses in `src/esp_safe.rs` must change (datasheet §4.1.2, Figure 4-1):

```
is_sane_sp():  0x3FFB_0000..0x4000_0000  →  0x3FC8_8000..0x3FF0_0000 [+ RTC fast/slow]
is_exec IRAM:  0x4000_0000..0x400C_2000  →  0x4037_0000..0x403E_0000
is_exec flash: 0x400D_0000..0x4040_0000  →  0x4200_0000..0x43FF_FFFF
UART0_BASE:    0x3FF4_0000               →  0x6000_0000
```

All four updated ✅

### Phase 4: Peripheral-specific changes

```rust
const _: () = assert!(RMT_CHUNK_MAX <= 192, "S3 RMT shared RAM = 384 words");
```

Added in `stepper.rs:46-49` ✅

**ADC** (§4.2.2.1, Table 5-6): DB_12 range = 0–2900 mV (vs 0–3900 mV on ESP32). Sampling rate = 100 kSPS (vs 200 kSPS). Recalibrate `adc_cal` coefficients. Updated in `adc.rs` and `adc_cal.rs` ✅

**BLE** (§4.3.3): No Bluetooth Classic. Removed `ESP_COEX_PREFER_BT` references. Dead `set_coex_ble_preferred()` removed ✅

**WiFi** (§4.3.2): WPA3 available as optional upgrade. No breaking changes.

**IRAM placement**: Panic handler chain placed in IRAM via `#[link_section = ".iram1"]` ✅

### Phase 7: Boot crash fix — `disable_wdt()` logs before `logger::init()`

**Root cause**: PR #1 (disable_wdt return value check) introduced `log::warn!()` into `disable_wdt()`, which was called BEFORE `logger::init()`. When `esp_task_wdt_deinit()` fails on S3, the early `log::warn!()` panics.

**Fix applied**: `logger::init()` moved before `disable_wdt()` in `main.rs:87,89` ✅
